Integration with Coinbase AgentKit: USDC Giveaway Payouts

We have a “No-Code Social Agent Launcher” project (React + Express + TypeScript + PostgreSQL), which includes:

1. A Wizard UI to create social agents (Poll/Q&A/Giveaway).
2. A Telegram bot that handles user commands for polls/giveaways.
3. A PostgreSQL DB to store user, poll, and giveaway data.

We want to integrate Coinbase AgentKit so our giveaways can *actually* pay winners in USDC. Here’s what we need:

----------------------------------------------------------------------------------
1) Install AgentKit
----------------------------------------------------------------------------------
- Add “@coinbase/agentkit” (or the correct library name) to our server `package.json`.
- We’ll store credentials (API key/secret, etc.) in environment variables, e.g., `COINBASE_AGENTKIT_API_KEY`.

----------------------------------------------------------------------------------
2) Basic AgentKit Setup
----------------------------------------------------------------------------------
- Create a new file: `server/src/coinbase/agentKit.ts`
  - Import the AgentKit package.
  - Initialize an MPC wallet or the relevant object with our environment variables.
  - Export a simple function, e.g. `async function sendUsdc(toAddress: string, amount: string): Promise<void>`, which uses AgentKit to send USDC from the agent’s wallet to `toAddress`.

----------------------------------------------------------------------------------
3) Tie into Giveaway Logic
----------------------------------------------------------------------------------
- In `giveawayService.ts` (where `pickGiveawayWinner` is implemented):
  - After we pick a winner (e.g., random from `giveaway_entries`), call `sendUsdc` with the winner’s on-chain address and a fixed or dynamic amount.
  - We'll need a place to store each user's wallet address. For now:
    - Add a `walletAddress` column to `users` or create a new `user_addresses` table.
    - When a user enters a giveaway, we can retrieve or prompt them for their wallet address.
  - If the user has no wallet address, we skip the payout or show an error.

----------------------------------------------------------------------------------
4) Frontend UI Updates
----------------------------------------------------------------------------------
- Add a “Wallet Address” field to the user registration or profile screen so we can store it in the DB.
- Display the agent's or brand’s on-chain balance (optional). We can add a “Check agent’s USDC balance” button that calls a `getBalance()` method in `agentKit.ts`.
